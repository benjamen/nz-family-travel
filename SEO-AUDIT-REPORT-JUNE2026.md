# NZ Family Travel — SEO Audit Report

**Date:** 13 June 2026
**Auditor:** SEO specialist agent (seo-scan-team)
**Site:** https://nzfamilytravel.co.nz
**Build:** 399 HTML pages (was 396 pre-audit)

---

## Summary

The site was already in strong technical shape: `sitemap.xml`, `robots.txt`, and `llms.txt` are all generated by `build.py`; `base.html` ships canonical, full Open Graph + Twitter Card tags, and WebSite + Organization JSON-LD on every page; section templates carry rich schema (Article, BreadcrumbList, TouristDestination, Product, TouristTrip, AggregateRating, etc.).

The audit focused on the gaps that remained. The highest-impact finding was a **FAQ schema bug**: the guide template only emitted `FAQPage` schema when content used the `faqs` key, but 17 of 27 guide/post files store FAQs under `faq` — so those pages had FAQ content with no schema. Separately, FAQs were **never rendered visibly** on guide pages (schema-only FAQs violate Google's structured-data policy). Both are now fixed, lifting FAQ coverage on travel-tips pages from ~10 (schema-only) to **30 (schema + visible)**.

## Fixes Applied

1. **FAQ schema normalisation** (`layouts/guide.html`) — added a template prelude that merges both `guide.faqs` (`{q,a}`) and `guide.faq` (`{question,answer}`) into one normalised `_faqs` list, and drives the `FAQPage` JSON-LD from it. FAQ schema now fires for all pages with FAQ content regardless of key.
2. **Visible FAQ section** (`layouts/guide.html` + `static/css/style.css`) — added a rendered "Frequently Asked Questions" `<details>` accordion above the bottom CTA, so structured FAQ data is backed by on-page content. Added `.faq-item .faq-answer` CSS to match the existing FAQ styling.
3. **Orphan hub internal links** (`layouts/base.html`) — added a "Plan Your Trip" footer block linking the six previously-unlinked section hubs (cities, overseas, weather, school-holidays, holiday-parks, nz-map). Applies site-wide (every page), resolving the orphan-hub gap in one change.
4. **CLS / image dimensions** — added `width`/`height` to all 5 template `<img>` tags (hero images 1200×630, destination gallery thumbs 600×220). Existing CSS already constrains rendered size, so attributes only supply aspect ratio for layout reservation.
5. **Meta description trimming** — trimmed 22 descriptions over 185 chars down to ≤160 at word boundaries (worst was 296 chars).
6. **Rebuild** — `python build.py` succeeds cleanly, 399 pages, 0 invalid JSON-LD blocks across audited pages.

## Content Stubs Created

All three fill genuine gaps identified in `content-strategy/strategy.md` (topics marked ❌ in its cluster map). They are full, build-valid guide JSON (title, meta_desc, focus_keyword, schema_type, intro, h2_outline, sections, faq, hero_image) and auto-appear in the travel-tips hub, footer, sitemap, and search index.

1. **`nz-with-tweens-travel-guide.json`** — "NZ with Tweens (10-13)" — focus keyword `nz holiday with 10 year old`. Fills the Pillar 1 age-specific gap (tweens were the only unbuilt age bracket).
2. **`nz-winter-school-holidays-activities.json`** — "NZ Winter School Holidays — Activities & Ideas" — focus keyword `nz winter school holidays`. Pillar 2 seasonal gap.
3. **`nz-easter-holiday-ideas-families.json`** — "NZ Easter Holiday Ideas for Families" — focus keyword `nz easter holiday ideas`. Pillar 2 seasonal gap.

> Stubs ship with placeholder hero image paths (`/static/img/destinations/<slug>.jpg`). The template falls back to a default OG image if the file is absent, so the build is unaffected — but real images should be added before promotion.

## Remaining Recommendations (human action)

1. **Flesh out the 3 stubs** — each has 4 outlined sections; expand to 7–8 sections with real 2026 prices/dates and add hero images to match the depth of existing guides (e.g. `skiing-nz-with-kids`).
2. **Title length tuning (~40 pages)** — titles of 61–67 chars get mid-word truncated in SERPs. Trimming to ≤60 is a low-effort polish; not done in this pass because it requires editorial judgement to keep the primary keyword. Worst offenders: `nz-road-trip-snacks-kids` (67), `jucy-vs-britz-campervan-comparison` (67), `nz-7-day-north-island-family-itinerary` (67).
3. **Short descriptions (~25 pages)** — several tool and travel-tip descriptions are 84–119 chars; expanding toward 150–160 with a CTA would use more SERP real estate.
4. **Affiliate placeholders** — `site.json` has `PENDING_SIGNUP` / `PENDING_*_ID` URLs for Booking.com, Viator, and World Nomads. These currently render live links that won't track commission. Resolve before relying on those revenue lines.
5. **Pin school-holiday dates** — the winter/easter stubs reference 2026 dates loosely ("early-to-mid July"); confirm against the school-holidays data source and hard-code exact dates for featured-snippet eligibility.
6. **Consider a top-nav "More" dropdown** — the footer now links the orphan hubs, but adding cities/weather/school-holidays to the primary nav would further strengthen their crawl priority.
